# ScoreManager 仕様書

## 1. 概要

ローカルに保存された PDF 楽譜を、演奏中に見やすい形で表示・操作するブラウザベースの楽譜ビューアを開発する。  
カテゴリ別フォルダ構成をそのまま本棚 UI に反映し、楽譜選択後は全画面リーダーへ遷移する構成とする。

関連動画 URL などの楽譜ごとの基本データは、選択したルートフォルダ内の JSON ファイルへ保存する。  
ブラウザ側には、次回復元用の軽量な状態だけを保持する。

## 2. 機能要件

### 2.1 楽譜ライブラリ

- ユーザーが選択したルートフォルダ以下を再帰的に走査し、カテゴリフォルダと PDF 楽譜を一覧表示できる
- 既存のフォルダ構成を壊さず、カテゴリ単位でたどれる
- PDF 楽譜はタイトルと 1 ページ目サムネイルを本棚風のカードで表示する
- フォルダを開くと中のフォルダや楽譜を同じ UI で閲覧できる

### 2.2 演奏ビュー

- 楽譜を単ページ表示・見開き表示・スクロール表示で切り替えられる
- リーダー画面は可能な限り楽譜の表示領域を優先する
- 常時表示の UI は最小限にし、詳細操作はメニューボタンから開く
- 右端 / 左端のタッチ操作で進む・戻るができる
- キーボード入力をページ送りとして扱える
- フットペダルがキーボードエミュレーションする場合、その入力をそのまま利用できる

### 2.3 自動ページ送り

- スクロール表示モードでのみ自動スクロールを利用できる
- BPM と 1 ページあたりの拍数を設定し、そこからスクロール速度を算出する
- 演奏中に開始 / 停止を切り替えられる

### 2.4 関連動画

- 楽譜ごとに外部動画 URL を保存できる
- 保存済み URL をワンクリックで別タブ表示できる
- YouTube に限らず通常の動画 URL も扱える

### 2.5 設定画面

- 常時表示しない設定専用ページを持つ
- 保存方式、現在のルートフォルダ、現在カテゴリなどを確認できる
- ルートフォルダを変更できる

## 3. 実装要件

- フロントエンドはプレーンな HTML / CSS / JavaScript で実装する
- PDF 描画には `pdf.js` を使用する
- ライブラリのフォルダ選択には File System Access API を使用する
- 対応ブラウザは Chromium 系ブラウザを主対象とする
- 楽譜ごとの基本データはルートフォルダ内の `.score-manager-data.json` に保存する
- current folder path など軽量状態は `localStorage` に保存する
- ルートフォルダへの再接続用ハンドルは `IndexedDB` に保存する

## 4. 基本設計

### 4.1 画面構成

1. 本棚ページ
2. 全画面リーダーページ
3. 設定ページ

### 4.2 データ構造

```text
ScoreManagerData {
  version: number,
  updatedAt: string | null,
  scores: {
    [relativePdfPath: string]: {
      videoUrl?: string
    }
  }
}
```

### 4.3 保存場所

- `.score-manager-data.json`: ルートフォルダ内
- `localStorage`: 現在開いているカテゴリ、表示中ページなど
- `IndexedDB`: ルートディレクトリハンドル

## 5. 詳細設計

### 5.1 楽譜の表示モード

- 単ページ: 1 ページだけ描画する
- 見開き: 左右に 2 ページを並べる
- スクロール: 全ページを縦に積んで表示する

### 5.2 ページ送り入力

- タッチゾーン左: 前へ
- タッチゾーン右: 次へ
- `ArrowLeft`, `PageUp`: 前へ
- `ArrowRight`, `PageDown`: 次へ
- `Space`: 自動スクロールの開始 / 停止

### 5.3 リーダー UI

- 上部に最小情報としてタイトルとページ番号のみ表示する
- メニューボタン押下時のみ、表示モード切替や動画 URL 入力を表示する
- 設定は別ページで扱う

## 6. 実装上の注意点

- File System Access API は `https` または `http://localhost` 系での起動が必要
- ルートフォルダ内へ JSON を書き込むため、読み取りだけでなく書き込み権限も必要
- PDF.js は CDN 参照のため、初回利用時はネットワーク接続が必要
- フットペダルの HID 直接連携は機種依存のため、第一段階ではキーボード互換入力を標準サポートとする

## 8. テストの実行方法

1. ローカルサーバーで `ScoreManager` を公開する
2. Chromium 系ブラウザで `http://127.0.0.1:8123` を開く
3. 楽譜フォルダを選択する
4. 動画 URL 保存後、ルートフォルダ直下に `.score-manager-data.json` が作られることを確認する
5. ページ再読込後に現在カテゴリが復元されることを確認する
